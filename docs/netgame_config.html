 Netgame - Config
Overview Settings you can change Settings you probably shouldn't change!
Overview There are a number of variables in the file 'pm.cfg' used to setup and control netgame. 
[NETWORK]
server_IP = 192.168.1.3
mNetgame.TCP=0
mPlayer.loc[0].server_state_freq_mode=1
mShot.deathmatch_shots=0
mShot.deathmatch_shot_damage=5
mShot.suicide_shots=0
Settings you can change
server_IP This is for the clients only. They need to know the server hostname or IP address to connect to. This value is completely ignored by the server. A server doesn't need to have anything set for clients. 
valid entries: 
server_IP=192.168.1.3
server_IP=myhostname
mShot.deathmatch_shots Used to set Deathmatch Mode. Controls whether player's shots affect other players. 
valid entries: 
mShot.deathmatch_shots=0 // Deathmatch Mode Off
mShot.deathmatch_shots=1 // Deathmatch Mode On

default: 
mShot.deathmatch_shots=0 // Deathmatch Mode Off
mShot.deathmatch_shot_damage Used to set the amount of damage player's shots do in Deathmatch Mode. Has no effect if Deathmatch Mode is off. The value is the amount of health lost when shot by another player. 100 will instantly kill. 50 will take 50 health...etc. Note: If set to zero, shots will not take any health, but will still cause the player to recoil. Note: If set to a negative value, shots will actually give health! 
valid entries: 
mShot.deathmatch_shot_damage=100
mShot.deathmatch_shot_damage=50
mShot.deathmatch_shot_damage=10
mShot.deathmatch_shot_damage=0
mShot.deathmatch_shot_damage=-10

default: 
mShot.deathmatch_shot_damage=5
mShot.suicide_shots Controls whether player's shots affect the same player that shot them. This can happen if a player shoots, then immediately takes a door to where the shot is headed. It can also happen when shooting upwards, then jumping on a spring. 
valid entries: 
mShot.suicide_shots=0 // suicide shots off
mShot.suicide_shots=1 // suicide shpts on

default: 
mShot.suicide_shots=0 // suicide shots off

Settings you probably shouldn't change! For the variables below, it is not recommended to change them from their default values.
mNetgame.TCP Used to set the packet type. Must be the same on client and server. Both seem to work about the same. (Your mileage may vary....) 
valid entries: 
mNetgame.TCP=0 // use UDP packet type
mNetgame.TCP=1 // use TCP packet type

default: 
mNetgame.TCP=0 // use UDP packet type
mPlayer.loc[0].server_state_freq_mode Lets the server automatically adjust how often to send states to clients. 
valid entries: 
mPlayer.loc[0].server_state_freq_mode=0 // manual
mPlayer.loc[0].server_state_freq_mode=1 // auto

default: 
mPlayer.loc[0].server_state_freq_mode=1 // auto
